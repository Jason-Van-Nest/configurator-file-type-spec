# Contributing to the Configurator File Type Specification

Thank you for your interest in contributing! Here's how you can help:

## Ways to Contribute

1. **Propose Changes**: Open an issue to discuss changes or improvements
2. **Submit Proposals**: Use pull requests to propose edits to the specification
3. **Share Examples**: Contribute example configurator files
4. **Report Issues**: Found a problem? Let us know!

## Process

1. Fork or create a branch
2. Make your changes
3. Submit a pull request with a clear description
4. Participate in review discussion
5. Changes are merged once approved

## Code of Conduct

- Be respectful and constructive
- Focus on the proposal, not the person
- Help others understand your perspective

## Questions?

Open an issue or start a discussion!
